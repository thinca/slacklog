私の記憶が正しければ、

今って vital の各モジュールをロードするのに autoload を使っていて、直接 source しない理由は対象ファイルの script id を取得する必要があるからで、以前は scriptnames 等から取ってたけど速度面により今の形式になっている

と言う感じだったと思うのだけど、今新しい vital loader で別の手法を検討している。
グローバル変数を使う方法。例えば g:vital_script_id と言う名前を予約しておいて、

1. モジュールロード前にもし変数が存在していたら退避。
2. ローダがモジュールを source する。モジュールの先頭に let g:vital_script_id = expand('<sfile>') 的なのを vitalize 時に仕込む
3. ローダは g:vital_script_id を読み込み、元の状態に復元する(退避していたら戻す、元々なかったら消す)

この方法に何か抜けや問題点があるか、聞きたい
