緩質問

https://async.rs/blog/stop-worrying-about-blocking-the-new-async-std-runtime/

ここで言っている最適化ってのは async 関数の中身を丸ごと別スレッドとかに自動で退避するって意味なので

// std::fs::read_to_string を async でラップするだけ
async fn read_to_string(path: impl AsRef<Path>) -> io::Result<String> {
    std::fs::read_to_string(path)
}

// hello1 内の各行で Await が効き、いい感じにグリーンスレッドが切り替わる
async fn hello1() -> Result<()> {
    let a = read_to_string("A").await?;
    let b = read_to_string("B").await?;
    let c = read_to_string("C").await?;
}

// hello2 全体でしか Await が効かないため、あまり意味がない
async fn hello2() -> Result<()> {
    let a = std::fs::read_to_string("A")?;
    let b = std::fs::read_to_string("B")?;
    let c = std::fs::read_to_string("C")?;
}

って感じで、既存の非 async なライブラリは IO 発生する最小単位で async でラップして、それを使うようにしないと意味があまりない



という認識であってますかね？
