quickrun で async Main なプログラムを動かしたかったので調べたメモ
• dotnet cli ではプロジェクトでない（ csproj ファイルを伴わない）単体 cs ファイルの実行はできなさそう
• csc.exe は -langversion オプションで言語バージョンを指定できるが、標準の csc.exe では対応言語バージョンが古いため async Main など最近の言語仕様に非対応
• https://www.nuget.org/packages/Microsoft.Net.Compilers/ に付属している csc.exe であれば -langversion:preview なども対応
• nupkg ファイルは単なる zip ファイルなので、ダウンロード・解凍して好きなところに配置し、tools\csc.exe のパスを quickrun で設定してあげて、 -langversion オプションも指定するようにすれば良い（デフォルトの対応言語バージョンも上がっているので async Main であれば -langversion 指定は不要）
